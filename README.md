# Docker Homelab: Self-Hosted Infrastructure

Масштабируемая среда для развертывания веб-сервисов, мониторинга и медиа-контента на базе Docker Compose. Конфигурация ориентирована на безопасность, изоляцию сетевых уровней и автоматизацию контроля ресурсов.

---

## Архитектура системы

Инфраструктура разделена на логические сегменты:
* **Edge/Proxy:** Traefik обеспечивает маршрутизацию трафика и терминацию SSL. Для безопасности используется Docker Socket Proxy, исключающий прямой доступ Traefik к `/var/run/docker.sock`.
* **Web Services:** Nginx для статики и Python-backend (web-api) с настроенными healthcheck-проверками.
* **Observability:** Стек Prometheus + Grafana для сбора метрик с хоста и контейнеров.
* **Media:** Сервер Jellyfin с ограниченными правами пользователя и лимитами ресурсов.

---

## Быстрый старт

1. **Подготовка окружения**
   Склонируйте репозиторий и создайте файл конфигурации:
   ```bash
   cp .env.example .env
   ```
   Отредактируйте `.env`, установив свои пароли и пути к данным (`DATA_ROOT`).

2. **Запуск стека**
   ```bash
   docker compose up -d
   ```

3. **Проверка состояния**
   ```bash
   docker compose ps
   ```

---

## Спецификация конфигурации

### Безопасность и сеть
* **Network Isolation:** База данных и внутренние API вынесены в `internal` сети. Прямой доступ извне возможен только через прокси.
* **Resource Limits:** Для каждого сервиса жестко заданы лимиты CPU и RAM в блоке `deploy`, что предотвращает деградацию всей системы при утечках памяти в одном контейнере.
* **Log Management:** Глобально настроена ротация логов (`max-size: 10m`), защищающая дисковое пространство от переполнения.

### Схема портов
| Сервис | Порт (Local) | Назначение |
| :--- | :--- | :--- |
| Traefik | 80 / 8080 | HTTP / Дашборд |
| Grafana | 3000 | Мониторинг |
| Jellyfin | 8096 | Медиа-сервер |

## Кастомизация бэкенда

Чтобы подключить собственный бэкенд, замените параметры в секции `web-api`:
1. Укажите путь к вашему Dockerfile в `build.context`.
2. Настройте правила маршрутизации в `labels` (Traefik).
3. Убедитесь, что ваш сервис подключен к сетям `proxy` (для доступа извне) и `backend` (для связи с БД).

## Обслуживание

**Обновление образов:**
```bash
docker compose pull
docker compose up -d
```

**Просмотр логов конкретного сервиса:**
```bash
docker compose logs -f [service_name]
```

**Безопасная остановка:**
```bash
docker compose down
```
