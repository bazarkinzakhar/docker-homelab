# Homelab Docker Infrastructure

Репозиторий содержит готовую архитектуру для развертывания домашнего сервера на базе Docker и Traefik. Система включает в себя прокси-сервер, веб-интерфейс, бэкенд на Python, базу данных и полноценный стек мониторинга.

-----

## Технологический стек

  * **Traefik 2.11** — обратный прокси и автоматический роутинг
  * **Nginx 1.25** — фронтенд сервер для статики
  * **FastAPI (Python 3.11)** — бэкенд сервер
  * **PostgreSQL 16** — реляционная база данных
  * **Prometheus & Grafana** — сбор метрик и визуализация состояния системы
  * **Jellyfin** — медиасервер для домашнего кинотеатра
  * **Node Exporter** — мониторинг ресурсов хост-системы

-----

## Системные требования

  * **Docker Desktop** для macOS или Windows
  * **Docker Engine** и **Docker Compose** для Linux
  * Минимум **4 ГБ** оперативной памяти для корректной работы всех сервисов
  * Свободные порты **80**, **8080** и **3000** на хост-машине

-----

## Подготовка системы

Перед запуском на macOS необходимо настроить доступ к Docker Socket

1.  Откройте **Settings** в Docker Desktop
2.  Перейдите в раздел **Advanced**
3.  Активируйте чекбокс **Allow the default Docker socket to be used**
4.  Примените настройки и дождитесь перезапуска

-----

## Быстрый старт

### 1\. Клонирование и структура

Создайте рабочую директорию и убедитесь что структура проекта соответствует следующей схеме

  * **config/** — папка с конфигурациями nginx и prometheus
  * **web/** — исходный код python бэкенда и Dockerfile
  * **data/** — директория для хранения баз данных и логов (создается автоматически)
  * **.env** — файл с вашими секретами и настройками
  * **docker-compose.yml** — основной файл описания инфраструктуры

### 2\. Настройка переменных окружения

Создайте файл **.env** в корневой папке проекта и заполните его данными

```env
DATA_ROOT=./data
POSTGRES_USER=admin
POSTGRES_PASSWORD=your_password
POSTGRES_DB=homelab
GRAFANA_ADMIN_PASSWORD=your_grafana_password
JELLYFIN_PUID=1000
JELLYFIN_PGID=1000
```

### 3\. Запуск инфраструктуры

Выполните команду в терминале из корневой папки проекта

```bash
docker compose up -d
```

## Использование сервисов

После успешного запуска сервисы доступны по следующим адресам

  * **http://localhost** — фронтенд (Nginx)
  * **http://localhost/api** — бэкенд (Python API)
  * **http://localhost:8080** — дашборд Traefik (просмотр состояния роутов)
  * **[http://grafana.localhost](https://www.google.com/search?q=http://grafana.localhost)** — панель мониторинга Grafana
  * **[http://media.localhost](https://www.google.com/search?q=http://media.localhost)** — медиасервер Jellyfin

## Развертывание своих приложений

Чтобы добавить свой сервис в эту архитектуру достаточно добавить его в **docker-compose.yml** и прописать лейблы для Traefik

```yaml
labels:
  - "traefik.enable=true"
  - "traefik.http.routers.myservice.rule=Host(`myservice.localhost`)"
  - "traefik.http.services.myservice.loadbalancer.server.port=80"
```

## Решение частых проблем

### Ошибка 404 при доступе к сервисам

Проверьте вкладку HTTP Routers в дашборде Traefik на порту 8080. Если список пуст, значит Traefik не может прочитать docker.sock. Убедитесь что выполнен пункт Подготовка системы

### Ошибка прав доступа к папке data

На Linux или macOS выполните команду `chmod -R 777 ./data` для предоставления контейнерам прав на запись логов и баз данных

### Бэкенд не отвечает по пути /api

Убедитесь что в вашем Python коде роуты прописаны с учетом префикса или используйте middleware StripPrefix в настройках Traefik как это реализовано в текущем конфиге

## Обслуживание

Для остановки всех систем используйте команду `docker compose down`
Для просмотра логов конкретного сервиса используйте `docker compose logs -f имя_сервиса`
Для полного сброса данных используйте `docker compose down -v` (внимание все данные в БД будут удалены)
